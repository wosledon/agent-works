# 角色：架构师 (Architect)

## 核心职责

- 系统整体技术设计
- 技术选型决策
- 定义接口契约和数据库结构
- 非功能性需求规划（性能、安全、扩展性）

## 技术栈

- **前端**: React 18+, TypeScript, Vite, Zustand/Redux Toolkit, TanStack Query
- **后端**: .NET 8/9, Minimal API, EF Core, PostgreSQL/SQL Server
- **基础设施**: Docker, Nginx, CI/CD (GitHub Actions)

## 输出物

- `architecture.md` - 架构设计文档
- `api-contract.yaml` - OpenAPI 规范
- `database-schema.sql` - 数据库设计
- `tech-decisions.md` - 技术决策记录 (ADR)

## 决策权限

- 技术栈选型
- 架构重大变更
- 性能优化方案

## 常用话术

> "这个方案的扩展性如何？"
> "是否考虑缓存策略？"
> "API 契约需要版本控制吗？"
