# Git 提交规范

## 提交频率

**禁止**：攒大量代码后再提交  
**要求**：小步快跑，频繁提交

| 场景 | 提交时机 |
|------|---------|
| 完成一个功能点 | 立即提交 |
| 修复一个 bug | 立即提交 |
| 重构一段代码 | 立即提交 |
| 工作 1-2 小时 | 必须提交 |
| 下班前 | 必须提交 |

## 提交信息格式

```
<type>: <subject>

<body> (可选)
```

**type 类型**:
- `feat`: 新功能
- `fix`: 修复
- `docs`: 文档
- `style`: 格式（不影响代码运行的变动）
- `refactor`: 重构
- `test`: 测试
- `chore`: 构建/工具

**示例**:
```
feat: 添加文章发布接口

- 实现 POST /api/articles
- 添加发布状态校验
- 更新 Swagger 文档
```

## 禁止事项

- ❌ 一次性提交 10+ 个文件改动
- ❌ 提交信息写 "update"、"fix" 等无意义内容
- ❌ 提交编译产物（bin/obj/node_modules）
- ❌ 提交敏感信息（密码、密钥）

## 工作流程

```
1. 写代码（1-2小时或一个功能点）
2. git add -A
3. git commit -m "feat: xxx"
4. git push
5. 重复
```

## 检查清单

每次提交前检查：
- [ ] .gitignore 已配置
- [ ] 编译产物未提交
- [ ] 提交信息清晰
- [ ] 敏感信息已移除
