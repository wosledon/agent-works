# 关键测试用例 (Critical Test Cases)

## 文章管理模块

### TC-POST-001: 发布文章
```
优先级: P0
类型: 功能测试
前置: 用户已登录且有发布权限

步骤:
1. 进入文章创建页面
2. 填写标题（必填）、内容（必填）
3. 选择分类和标签
4. 点击"发布"按钮

预期:
- 文章状态变为"已发布"
- 文章列表可见
- API 返回 201 Created
- 创建时间、作者信息正确
```

### TC-POST-002: 编辑文章
```
优先级: P0
类型: 功能测试

步骤:
1. 选择已发布文章
2. 修改标题和内容
3. 保存更改

预期:
- 文章更新成功
- 更新时间字段更新
- 历史版本可追踪
```

### TC-POST-003: 草稿自动保存
```
优先级: P1
类型: 功能测试

步骤:
1. 开始创建文章
2. 输入部分内容后等待 30 秒

预期:
- 内容自动保存为草稿
- 页面刷新后内容恢复
```

## 用户认证模块

### TC-AUTH-001: 用户注册
```
优先级: P0
类型: 功能测试

步骤:
1. 访问注册页面
2. 输入邮箱、用户名、密码（8+字符含数字字母）
3. 提交注册

预期:
- 注册成功，返回登录页
- 邮箱收到验证邮件
- 数据库新增用户记录（密码加密）
- 重复邮箱注册失败
```

### TC-AUTH-002: 用户登录
```
优先级: P0
类型: 功能测试

步骤:
1. 输入正确邮箱和密码
2. 点击登录

预期:
- JWT Token 返回
- Token 有效期设置正确
- 跳转到首页
```

### TC-AUTH-003: Token 过期处理
```
优先级: P1
类型: 功能测试

步骤:
1. 用户登录后等待 Token 过期
2. 访问需要认证的 API

预期:
- 返回 401 Unauthorized
- 刷新 Token 机制触发（如有）
- 或引导重新登录
```

### TC-AUTH-004: 密码安全验证
```
优先级: P0
类型: 安全测试

检查点:
- 密码最小长度 8 位
- 必须包含大小写字母
- 必须包含数字或特殊字符
- 登录失败 5 次后锁定账户 15 分钟
- 密码传输使用 HTTPS
```

## 评论系统模块

### TC-CMT-001: 发表评论
```
优先级: P0
类型: 功能测试

步骤:
1. 登录用户浏览文章
2. 在评论框输入内容
3. 提交评论

预期:
- 评论显示在文章下方
- 评论作者、时间正确
- API 返回 201
```

### TC-CMT-002: 回复评论
```
优先级: P1
类型: 功能测试

步骤:
1. 点击某条评论的"回复"
2. 输入回复内容
3. 提交

预期:
- 回复嵌套显示
- 层级关系正确
- 被回复者收到通知
```

### TC-CMT-003: 评论审核
```
优先级: P1
类型: 功能测试（管理员）

步骤:
1. 管理员登录后台
2. 查看待审核评论列表
3. 审核通过/拒绝

预期:
- 通过后评论前台可见
- 拒绝后评论隐藏并通知用户
```

### TC-CMT-004: XSS 防护
```
优先级: P0
类型: 安全测试

步骤:
1. 输入含脚本的内容: <script>alert('xss')</script>
2. 提交评论

预期:
- 脚本被转义，不执行
- 存储时 HTML 编码
- 页面显示为纯文本
```

## API 接口模块

### TC-API-001: 获取文章列表
```
优先级: P0
类型: API 测试

请求: GET /api/posts?page=1&size=10

验证点:
- 返回 200 OK
- 分页参数正确
- 响应包含: id, title, excerpt, author, createdAt
- 按时间倒序排列
```

### TC-API-002: 未认证访问保护
```
优先级: P0
类型: API 测试 / 安全

请求: POST /api/posts (无 Token)

验证点:
- 返回 401 Unauthorized
- 返回 JSON 错误信息
- 数据库无新记录
```

### TC-API-003: 数据校验
```
优先级: P1
类型: API 测试

请求: POST /api/posts
Body: { "title": "", "content": "" }

验证点:
- 返回 400 Bad Request
- 错误信息指出具体字段
- 模型验证失败详情返回
```

### TC-API-004: 并发创建
```
优先级: P1
类型: 性能测试

场景: 100 用户同时创建文章

验证点:
- 所有请求正常处理
- 无数据冲突
- 响应时间 P95 < 500ms
- 数据库连接池正常
```

## 端到端用户旅程

### TC-E2E-001: 完整发布流程
```
优先级: P0
类型: E2E 测试 (Playwright)

场景:
用户注册 → 登录 → 创建文章 → 发布 → 查看文章

验证点:
- 各页面跳转正确
- 数据持久化
- 文章列表可见新文章
- 文章详情页显示正确
```

### TC-E2E-002: 评论互动流程
```
优先级: P1
类型: E2E 测试

场景:
用户A发布文章 → 用户B注册登录 → 用户B评论 → 用户A回复

验证点:
- 评论实时显示
- 回复通知送达
- 嵌套评论层级正确
```

## 数据库测试

验证点:

类型: 集成测试
