# 验收标准 - DotNet Blog

## 验收标准模板

每个功能验收标准包含以下维度：
- **功能完整性**：功能是否按需求实现
- **数据准确性**：数据处理是否正确
- **用户体验**：界面交互是否符合预期
- **性能表现**：响应时间是否达标
- **安全性**：是否存在安全漏洞

---

## 1. 文章发布模块

### AC-01 文章创建

| 验收项 | 标准 | 验收方式 |
|-------|------|---------|
| 标题输入 | 必填，长度 1-200 字符 | 单元测试 + 人工测试 |
| 正文编辑 | 支持 Markdown 语法 | 人工测试 |
| 封面图片 | 可选，支持 jpg/png/gif，最大 5MB | 单元测试 + 人工测试 |
| 保存草稿 | 点击保存后状态变为"草稿" | 集成测试 |
| 发布文章 | 点击发布后状态变为"已发布" | 集成测试 |

**边界条件：**
- 标题为空时提示错误
- 正文为空时允许保存草稿，但不允许发布
- 图片格式不支持时提示错误
- 图片超过大小限制时提示错误

### AC-02 文章编辑

| 验收项 | 标准 | 验收方式 |
|-------|------|---------|
| 加载现有内容 | 编辑时正确加载原文章内容 | 单元测试 |
| 更新保存 | 保存后数据正确更新到数据库 | 集成测试 |
| 发布时间 | 更新已发布文章时，更新时间变化 | 单元测试 |

### AC-03 草稿自动保存

| 验收项 | 标准 | 验收方式 |
|-------|------|---------|
| 自动保存间隔 | 编辑 30 秒后自动触发保存 | 人工测试 |
| 恢复草稿 | 重新编辑时提示恢复草稿 | 人工测试 |
| 草稿清理 | 发布后自动清理相关草稿 | 单元测试 |

## 6. 通用验收标准

### AC-14 响应式设计

| 验收项 | 标准 | 验收方式 |
|-------|------|---------|
| 桌面端 | 宽度 ≥ 1024px 时正常显示 | 人工测试 |
| 平板端 | 768px ≤ 宽度 < 1024px 时正常显示 | 人工测试 |
| 移动端 | 宽度 < 768px 时正常显示 | 人工测试 |

| 浏览器 | 版本要求 | 验收方式 |

| Firefox | 最近 2 个版本 | 人工测试 |
| Safari | 最近 2 个版本 | 人工测试 |
| Edge | 最近 2 个版本 | 人工测试 |

### AC-16 性能标准

| 验收项 | 标准 | 验收方式 |
|-------|------|---------|
| 首屏加载 | ≤ 2 秒 | 性能测试 |
| API 响应 | ≤ 500ms (P95) | 性能测试 |
