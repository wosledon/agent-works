## 📌 Phase 1: MVP (v0.1) - 基础可用

**目标**: 核心功能跑通，能设计简单报表并导出
**时间**: 2026年3月 - 4月

### 关键里程碑

- [x] 项目启动 & 架构设计
- [ ] 后端基础框架搭建
- [ ] 数据库连接管理模块
- [ ] 报表设计器前端（拖拽+属性面板）
- [ ] 基础组件（文本、图片、表格）
- [ ] 流式 PDF 导出
- [ ] 流式 Excel 导出
- [ ] 报表存储与版本管理

### 技术验证点

| 验证项 | 目标 | 状态 |
|-------|------|------|
| 流式 PDF 内存占用 | < 512MB @ 10万行 | 🔄 |
| 设计器渲染性能 | 50+ 组件不卡顿 | 🔄 |
| 数据库连接池 | 支持100+并发查询 | 🔄 |

## 📊 技术债务与优化

| 优先级 | 项目 | 计划版本 | 说明 |

| P0 | 前端性能优化 | v0.9 | 虚拟滚动、组件懒加载 |

---

## 🎯 路线图决策原则

1. **价值优先**: 优先开发用户高频使用的功能
2. **质量底线**: 不牺牲代码质量和性能来赶进度
3. **反馈驱动**: 根据早期用户反馈调整优先级
4. **技术可行性**: 确保技术方案能支撑功能实现
