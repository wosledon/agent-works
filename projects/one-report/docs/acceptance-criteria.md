# One-Report 验收标准

## 1. 性能验收标准（核心指标）

### 1.1 报表设计器性能

| 指标项 | 目标值 | 测试方法 | 验收标准 |
|--------|--------|---------|---------|
| 设计器首屏加载时间 | ≤ 3秒 | Lighthouse / 手动计时 | 在100Mbps网络下，从输入URL到可交互时间 ≤ 3s |
| 组件拖拽响应延迟 | ≤ 100ms | 浏览器Performance API | 从鼠标按下到组件开始移动 ≤ 100ms |
| 画布缩放流畅度 | ≥ 30fps | Chrome DevTools FPS | 缩放操作帧率不低于30fps，无明显卡顿 |
| 百个组件渲染时间 | ≤ 500ms | 自动化测试 | 画布包含100个组件时，完整渲染 ≤ 500ms |
| 撤销/重做响应 | ≤ 200ms | 手动测试 | 点击撤销按钮后状态恢复 ≤ 200ms |

### 1.2 数据查询性能

| 指标项 | 目标值 | 测试方法 | 验收标准 |
|--------|--------|---------|---------|
| 简单查询响应 (<1万行) | ≤ 1秒 | 数据库日志 | SELECT * FROM table LIMIT 10000 执行+传输 ≤ 1s |
| 复杂查询响应 (<10万行) | ≤ 5秒 | 数据库日志 | 含JOIN/GROUP BY的查询 ≤ 5s |
| 查询结果预览加载 | ≤ 2秒 | 手动测试 | 点击预览到首屏数据显示 ≤ 2s |
| 数据连接池获取连接 | ≤ 50ms | 连接池监控 | 从连接池获取连接平均耗时 ≤ 50ms |

### 1.3 导出性能（核心亮点）

#### 1.3.1 Excel 导出性能

| 数据规模 | 目标耗时 | 内存占用 | 测试数据 |
|----------|---------|---------|---------|
| 1万行 × 20列 | ≤ 2秒 | ≤ 100MB | 正常文本数据 |
| 10万行 × 20列 | ≤ 10秒 | ≤ 200MB | 正常文本数据 |
| 100万行 × 20列 | ≤ 60秒 | ≤ 500MB | 正常文本数据 |
| 100万行 × 50列 | ≤ 120秒 | ≤ 1GB | 含公式和样式 |

**关键验收项：**
- ✅ 导出过程中系统内存占用保持稳定，无OOM
- ✅ 支持SXSSF（流式Excel）模式
- ✅ 导出进度可实时查询（每10%更新一次）
- ✅ 导出失败时提供明确错误信息和部分结果

#### 1.3.2 PDF 导出性能

| 数据规模 | 目标耗时 | 内存占用 | 测试数据 |
|----------|---------|---------|---------|
| 10页报表 | ≤ 3秒 | ≤ 150MB | 图文混排 |
| 100页报表 | ≤ 15秒 | ≤ 200MB | 表格为主 |
| 1000页报表 | ≤ 120秒 | ≤ 500MB | 大数据表格 |

**关键验收项：**
- ✅ 矢量图形渲染清晰
- ✅ 分页断行正确处理（表格不跨页断裂）
- ✅ 嵌入字体文件大小合理

### 1.4 流式处理性能（架构核心）

| 场景 | 目标指标 | 验收标准 |
|------|---------|---------|
| 数据读取 | 流式游标 | 数据库查询使用游标，不一次性加载到内存 |
| 数据处理 | 分页处理 | 每批次处理 ≤ 5000 条记录 |
| 内存稳定性 | O(1)复杂度 | 处理100万行数据时，内存增长不超过初始值的200% |
| 并发导出 | 资源隔离 | 10个并发导出任务，各自内存独立不相互影响 |

## 6. 测试用例示例

### 6.1 性能测试用例：百万行Excel导出

```gherkin
场景: 导出百万行数据到Excel
  给定 数据库中有100万行销售数据
  当 用户创建报表并选择该数据源
  并且 点击导出Excel按钮
  那么 导出任务在60秒内完成
  并且 过程中系统内存占用不超过500MB
  并且 生成的文件大小在合理范围内
  并且 文件可以用Excel正常打开
  并且 数据行数正确（100万行）
```

### 6.2 功能测试用例：可视化设计器

```gherkin
场景: 拖拽组件设计报表
  给定 用户已登录并进入设计器
  当 从组件库拖拽表格到画布中央
  并且 拖拽调整表格大小到500x300
  并且 绑定数据源"销售数据"
  那么 画布上显示表格组件
  并且 表格预览显示数据
  并且 属性面板显示表格配置项
```
