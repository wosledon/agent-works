# One-Report 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 产品名称
**One-Report** - 一站式低代码报表设计与导出平台

### 1.2 产品愿景
让数据报表的创建、设计、导出变得简单高效，无需编写代码即可实现复杂报表需求。

### 1.3 目标用户
- **数据分析师**：需要快速生成业务报表
- **产品经理**：需要定制化数据看板
- **企业IT部门**：需要统一的报表解决方案
- **业务人员**：需要自助式报表生成能力

### 1.4 核心价值主张
- 🎨 **可视化设计**：拖拽式报表设计，所见即所得
- 🔌 **多源接入**：统一对接多种数据源
- 🚀 **高性能导出**：秒级生成大文件报表
- 💾 **低资源消耗**：流式处理，内存友好

## 2. 功能需求

### 2.1 低代码可视化报表设计器

#### 2.1.1 核心功能
| 功能模块 | 描述 | 优先级 |
|---------|------|--------|
| 画布设计区 | 无限画布，支持网格对齐、缩放、多选 | P0 |
| 组件库 | 表格、图表、文本、图片、容器等基础组件 | P0 |
| 属性面板 | 组件样式、数据绑定、事件配置 | P0 |
| 图层管理 | 组件层级调整、显示/隐藏、锁定 | P1 |
| 模板市场 | 预置报表模板，支持导入导出 | P1 |
| 版本历史 | 报表版本管理，支持回滚 | P2 |

#### 2.1.2 组件详细清单

**基础组件**
- 文本标签（支持富文本、动态表达式）
- 图片（支持URL/上传/Base64）
- 容器（面板、卡片、分组框）
- 分割线、占位符

**数据组件**
- 数据表格（支持分页、排序、筛选）
- 交叉报表（行列透视）
- 明细列表
- 汇总卡片

**图表组件**
- 柱状图/条形图（分组、堆叠）
- 折线图/面积图（多系列）
- 饼图/环形图/玫瑰图
- 散点图/气泡图
- 雷达图
- 地图（热力图、区域图）
- 仪表盘/进度条

**高级组件**
- 数据透视表
- 树形表格
- 甘特图
- 自定义HTML

#### 2.1.3 交互功能
- 拖拽调整大小和位置
- 键盘快捷键（复制/粘贴/删除/撤销）
- 对齐辅助线
- 组件组合/解组
- 条件格式（根据数据动态改变样式）

---

### 2.2 多数据源支持

#### 2.2.1 数据源类型

| 数据源类型 | 支持方式 | 优先级 |
|-----------|---------|--------|
| **关系型数据库** | JDBC直连（MySQL、PostgreSQL、Oracle、SQL Server、SQLite） | P0 |
| **NoSQL数据库** | MongoDB、Elasticsearch、Redis | P1 |
| **大数据引擎** | ClickHouse、Apache Hive、Presto/Trino | P1 |
| **HTTP API** | RESTful API、GraphQL | P0 |
| **文件存储** | Excel、CSV、JSON、Parquet（本地/云存储） | P0 |
| **消息队列** | Kafka（流式数据源） | P2 |
| **云服务** | AWS S3、阿里云OSS、腾讯云COS | P1 |

#### 2.2.2 数据源配置
- 连接池管理（最大连接数、超时配置）
- 连接测试与健康检查
- 凭据加密存储
- 数据源分组与权限控制
- 连接复用与缓存策略

#### 2.2.3 数据查询
- 可视化查询构建器（QBE）
- SQL编辑器（语法高亮、自动补全、参数绑定）
- 参数化查询（支持日期范围、下拉选择等）
- 数据预览（前N条记录）
- 查询结果缓存（TTL配置）

#### 2.2.4 数据转换
- 字段映射与重命名
- 数据类型转换
- 数据过滤与清洗
- 数据聚合（分组、汇总）
- 数据合并（Join/Union）

---

### 2.3 高性能导出引擎

#### 2.3.1 导出格式支持

| 格式 | 特性 | 优先级 |
|------|------|--------|
| **PDF** | 矢量渲染、分页控制、书签导航 | P0 |
| **Excel (.xlsx)** | 大文件支持、公式保留、样式保留 | P0 |
| **Excel (.xls)** | 兼容旧版本 | P1 |
| **CSV** | 纯文本导出 | P0 |
| **Word (.docx)** | 报告导出 | P1 |
| **PPT** | 演示文稿导出 | P2 |
| **HTML** | 网页发布 | P1 |
| **图片 (PNG/JPG)** | 截图导出 | P1 |
| **JSON** | 数据接口 | P1 |

#### 2.3.2 导出配置
- 页面设置（纸张大小、方向、边距）
- 页眉页脚（动态内容支持）
- 水印设置（文字/图片水印）
- 密码保护（PDF加密）
- 压缩选项

#### 2.3.3 导出模式
- **同步导出**：小文件即时下载
- **异步导出**：大文件后台处理，完成后通知
- **定时导出**：按CRON表达式自动生成
- **批量导出**：多报表合并导出

---

### 2.4 流式处理与低内存架构

#### 2.4.1 核心技术特性
- **流式数据读取**：数据库游标、分页查询，避免全量加载
- **流式渲染**：边读取边生成，O(1)内存复杂度
- **分块处理**：大数据集分批次处理
- **内存池管理**：复用缓冲区，减少GC压力

#### 2.4.2 性能优化策略
- 数据预处理（索引优化、查询优化）
- 并行导出（多线程/协程）
- 增量更新（仅导出变化数据）
- 结果缓存（热点数据缓存）

## 3. 系统架构需求

### 3.1 技术栈建议
```
前端: React/Vue + TypeScript + Canvas/SVG
后端: Go/Java + 流式处理框架
存储: PostgreSQL + Redis + 对象存储
导出引擎: Apache POI (Excel) + iText/PDFBox (PDF)
```

### 3.2 部署架构
- 容器化部署（Docker + K8s）
- 水平扩展支持
- 负载均衡
- 服务降级与熔断

### 3.3 安全需求
- HTTPS/TLS 传输加密
- 数据脱敏（敏感字段掩码）
- 行级数据权限
- 操作审计日志
- SQL注入防护
